In-Canvas UI Hierarchy Demo

Description [QAD]: Demonstrates basic hierarchical (i.e. 'tree', 'recursive') structure for UI objects; e.g. "a button within a sub-menu within a menu within the UI partition of the canvas". The main reason for this design is spatial partitioning, i.e. handling an 'On Mouse Pressed' event via traversing through the tree starting at the root UIContainer, via the mouse coordinates thereof.
